# AI代理浏览器自动化

**一句话总结：** AI 智能体正在从「能读能写代码」进化到「能像人一样操作网页」——这意味着你的 AI 助手即将学会自己订机票、填表单、抓数据。

## 背景：为什么 AI 需要浏览器？

过去两年，AI 编程助手的能力飙升。Claude Code 能写代码、跑测试、提交 PR；ChatGPT 能分析文档、生成报告。但有一类任务它们始终做不好：**和网页打交道**。

想让 AI 帮你在携程上搜航班？抱歉，它只能给你一个链接。想让 AI 自动填写政府网站的申报表？它会告诉你「我无法直接操作网页」。

问题出在哪？

传统 AI 助手的能力边界是 **API 和文件系统**。它们能调用 Slack API 发消息、能读写本地文件、能执行 shell 命令。但网页是另一回事——网页是为人类设计的视觉界面，没有标准化的「API」可以调用。

于是就有了一个尴尬的局面：AI 能帮你写出一个完整的电商网站，却不能帮你在淘宝上比个价。

这正是「浏览器自动化」要解决的问题。

## 发生了什么

2026 年 2 月，GitHub Trending 上同时出现了两个项目：

**browser-use/browser-use**

这是一个 Python 库，口号是「让网页对 AI 智能体可访问」。它的核心思路是：把网页渲染成一种 AI 能理解的结构化格式，然后让 AI 决定点击哪里、输入什么。

技术上，它做了几件事：
- 用 Playwright 控制浏览器（支持 Chrome、Firefox）
- 把 DOM 元素转换成带坐标和语义标签的「视觉地图」
- 让 LLM（支持 GPT-4、Claude、Gemini 等）基于这个地图生成操作指令
- 自动处理弹窗、验证码、页面跳转等边缘情况

一句话：它是 AI 智能体的「眼睛」和「手」。

**vercel-labs/agent-browser**

Vercel 实验室出品的命令行工具，定位更明确——给 AI 智能体用的浏览器自动化 CLI。

它的特点是：
- 无头运行（headless），适合服务器部署
- 原生支持 AI SDK（Vercel 自家的 ai 包）
- 强调「代理优先」（Agent-first）设计
- 内置截图、元素定位、表单填写等原语

两个项目的共同点：都在试图建立一个标准化的接口层，让 AI 能像人类一样「看到」网页、「操作」网页。

## 这事为什么重要

### 1. 补上 AI 能力版图的最后一块

AI 智能体目前的能力版图大概是这样的：

| 能力域 | 成熟度 | 代表工具 |
|-------|-------|---------|
| 代码生成与编辑 | ⭐⭐⭐⭐⭐ | Claude Code, Cursor |
| 文件读写 | ⭐⭐⭐⭐⭐ | 几乎所有 AI 助手 |
| API 调用 | ⭐⭐⭐⭐ | MCP 协议, Function Calling |
| 数据库查询 | ⭐⭐⭐⭐ | Text-to-SQL |
| **网页操作** | ⭐⭐ | browser-use, agent-browser |

网页操作是明显的短板。而这块短板限制了一大类真实场景：

- 竞品监控（每天自动抓取对手定价）
- 政务申报（自动填写各种表单）
- 数据采集（从没有 API 的网站获取信息）
- 流程自动化（RPA 类任务）

补上这块，AI 助手的「可做事项」将大幅扩展。

### 2. 从「工具调用」到「环境交互」的范式转换

传统的 AI 工具调用是**声明式**的：你告诉 AI「调用天气 API，参数是北京」，AI 返回结构化数据。

浏览器自动化是**交互式**的：AI 需要「看」页面、「判断」当前状态、「决定」下一步操作、「验证」操作结果。这是一个完整的感知-决策-执行循环。

这比简单的 API 调用复杂得多：

- **状态不确定**：网页可能加载慢、可能弹出广告、可能需要登录
- **反馈延迟**：点击后要等待页面响应，而不是立即返回
- **视觉理解**：很多信息（按钮位置、颜色暗示、布局逻辑）只存在于视觉层

这推动 AI 向更「代理化」（agentic）的方向演进。不是执行单条指令，而是自主导航一个动态环境。

### 3. 对 RPA 行业的冲击

传统 RPA（机器人流程自动化，Robotic Process Automation）是一个价值数百亿美元的市场。UiPath、Automation Anywhere 这些公司靠的就是帮企业自动化网页和桌面操作。

但传统 RPA 有两个痛点：

1. **开发成本高**：需要手动录制或编写操作脚本，每个流程都是定制开发
2. **维护成本高**：网页一改版，脚本就废了

AI 驱动的浏览器自动化直接解决这两个问题：

- 不需要写脚本，用自然语言描述任务即可
- 网页改版？AI 能「看懂」新界面，自动适应

这不是渐进式改进，是范式颠覆。传统 RPA 厂商要么转型（把 LLM 整合进产品），要么被淘汰。
