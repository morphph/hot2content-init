## 什么是函数调用（Function Calling）？

函数调用（Function Calling），也叫工具使用（Tool Use），是让大语言模型突破纯文本生成限制、调用外部工具和函数的核心能力。模型不再仅仅输出自然语言，而是可以生成结构化的请求——通常是 JSON 格式——指定要调用的函数名称和参数。应用程序执行该函数后，将结果返回给模型。

举个例子，当用户问"东京现在天气怎么样？"时，具备函数调用能力的模型会生成 `get_weather(location="Tokyo")` 这样的结构化调用，获取实时数据后，再用自然语言组织出包含准确信息的回答。

## 工作原理

函数调用遵循清晰的工作流程：

1. **工具定义**：开发者向模型提供可用函数列表，每个函数包含名称、描述和参数结构
2. **模型决策**：模型根据用户请求判断是否需要调用函数、调用哪个函数，还是直接回答
3. **结构化输出**：模型生成 JSON 对象，指定函数名和参数
4. **执行调用**：应用程序执行函数（API 请求、数据库查询、计算等）
5. **结果整合**：函数结果返回给模型，模型将其整合为最终回答

现代模型支持顺序调用多个函数（链式调用）或并行调用，从而实现复杂的多步骤工作流。

## 为什么函数调用很重要？

函数调用是 2026 年 AI Agent 的基石：

- **实时数据**：模型可以访问最新信息，不再受限于训练数据的截止日期
- **执行操作**：AI Agent 可以发送邮件、创建文件、更新数据库、控制系统
- **提高准确性**：计算、查询和数据检索由可靠的工具完成，而非模型凭空生成
- **可组合性**：通过组合多个工具，模型可以处理任意复杂的任务
- **MCP 标准化**：模型上下文协议（MCP）统一了工具接口，使函数调用在不同模型和平台间通用

目前所有主流 AI 平台——Claude、GPT、Gemini——都原生支持函数调用，它已成为 AI Agent 与现实世界交互的主要机制。

## 相关术语

- **MCP（模型上下文协议）**：统一函数调用接口的开放标准
- **Agentic Coding（智能体编程）**：基于函数调用实现自主任务完成的开发范式
- **Agent Teams（智能体团队）**：多个 Agent 通过函数调用协作完成复杂工作流
