## 什么是提示工程？

提示工程是设计输入（提示词）来引导语言模型产生准确、有用且格式良好的输出的技术与实践。它是开发者和用户在不修改模型本身的情况下自定义 AI 行为的主要方式。

最简单的形式是写清晰的指令。最高级的形式涉及结构化模板、动态上下文注入和系统化的提示组件优化。

## 核心技术

提示工程包含一系列经验证的技术：

- **系统提示**：预先设定模型的角色、约束和输出格式
- **少样本示例**：提供 2-5 个输入-输出示例来演示期望模式
- **思维链（CoT）**：要求模型"逐步思考"以完成复杂推理任务
- **XML/结构化分隔符**：使用 `<context>`、`<instructions>` 等标签组织复杂提示
- **角色提示**："你是一位资深 Python 开发者"以激活领域知识
- **输出格式化**：明确指定 JSON、Markdown 或其他结构化输出格式

## 为什么重要

提示工程往往决定了 AI 应用是平庸还是卓越：

- **无需训练**：无需微调成本或数据收集即可即时定制行为
- **快速迭代**：在分钟内测试和改进提示，而非数小时/天
- **模型无关**：好的提示原则适用于 Claude、GPT、Gemini 等各种模型
- **成本控制**：更好的提示可以用更小、更便宜的模型达到相同质量

2026 年，提示工程已从小众技能成长为开发者的核心能力。有了百万 token 上下文窗口，提示工程现在还包括设计如何结构化和呈现大量上下文——不仅仅是编写指令。Claude Code 的 Skills 和 CLAUDE.md 文件本质上就是产品化的提示工程。

## 相关术语

- **思维链**：一种用于改善推理的提示技术
- **上下文窗口**：提示和上下文可用的空间
- **微调**：修改模型权重的替代定制方式
