# 如何使用 Claude Code 智能搜索：常见问题解答

### 1. Claude Code 的智能搜索和普通搜索有什么区别？

传统搜索只是把网页结果原样丢给你，而 Claude Code 的智能搜索（Agentic Search）会主动思考、过滤、执行代码来处理搜索结果。根据 Anthropic 最新发布的数据，Sonnet 4.6 在启用智能搜索后，BrowseComp 基准测试准确率提升了 13%，同时输入 token 消耗减少了 32%。这意味着它不只是"搜到了"，而是"搜对了"——在结果进入上下文窗口之前，AI 已经帮你筛掉了噪音。

### 2. 怎么在 Claude Code 里启用智能搜索功能？

智能搜索是 Claude Code 的内置能力，不需要额外配置。当你在对话中提出需要外部信息的问题时，Claude 会自动判断是否需要调用 Web Search 或 Web Fetch 工具。你也可以直接说"帮我搜索一下 XXX 的最新进展"来明确触发。如果你在使用 API，确保启用了相关工具权限即可。

### 3. 智能搜索能帮我做代码调研吗？比如找某个库的最新用法？

这正是智能搜索的强项。假设你想了解 React 19 的新特性，Claude Code 会搜索最新文档和博客、过滤过时的 React 18 内容、提取关键 API 变更，然后给你一份结构化的总结。它甚至会执行代码来验证语法是否正确。相比手动翻 10 篇博客，效率提升显著。

### 4. 搜索结果太多会不会把上下文窗口撑爆？

不会。这也是 Anthropic 这次更新的重点——智能搜索会在结果进入上下文之前就进行过滤和压缩。根据官方数据，token 消耗减少了 32%。它会写代码来提取关键信息、丢弃无关内容，而不是把整个网页塞进对话。你可以把它理解为一个会帮你做笔记的研究助理。

### 5. 我可以让 Claude Code 搜索特定网站的内容吗？

可以。你只需要在提问时指定来源，比如"帮我在 GitHub 上搜索 Rust async runtime 的实现"或"从 MDN 上查一下 Fetch API 的错误处理"。Claude 会优先从你指定的域名获取信息。对于需要登录的网站，目前智能搜索无法访问，但你可以手动提供链接让它抓取公开内容。

### 6. 智能搜索支持中文搜索吗？结果准确吗？

支持。Claude Code 可以处理中文查询，也能理解中文网页内容。不过由于中文互联网的信息质量参差不齐，建议在搜索技术内容时用英文关键词，或者明确指定权威来源（如官方文档、知名技术博客）。如果搜索结果不理想，可以让 Claude 用英文重新搜一次，然后用中文给你总结。

### 7. 智能搜索和提示词缓存（Prompt Caching）怎么配合使用？

这是目前最高效的组合。提示词缓存可以存储系统提示和常用上下文，而智能搜索负责获取实时信息。两者结合后，你既不用每次都重复发送大段背景信息，又能保证回答基于最新数据。正如开发者社区里说的："agentic search and prompt caching rule everything around me"——这两个功能叠加后，响应速度和成本效率都有明显改善。

### 8. 能用智能搜索来跟踪 GitHub Issue 或 PR 吗？

可以，但有更好的方式。Claude Code 内置了 `gh` CLI 工具，可以直接执行 `gh issue view` 或 `gh pr view` 命令获取结构化数据。如果你只是想了解某个开源项目的最近动态，智能搜索可以帮你抓取 Release Notes 或博客公告。两种方式可以组合使用：用 `gh` 查具体 Issue，用搜索查社区讨论和相关文章。

### 9. 智能搜索会不会返回过时或错误的信息？

AI 搜索确实存在这个风险，但 Claude Code 做了几层防护。首先，它会优先选择近期发布的内容；其次，它会交叉验证多个来源；最后，对于代码相关的结论，它可能会尝试执行来验证正确性。如果你对某个答案存疑，可以要求 Claude 给出信息来源，它会提供原始链接供你核实。

### 10. 有什么技巧能让智能搜索的结果更精准？

三个建议：一是问题要具体，"Python 怎么处理日期"不如"Python 3.12 如何用 datetime 解析 ISO 8601 格式"；二是提供上下文，比如"我在用 FastAPI 开发，想知道..."；三是明确期望格式，比如"给我一个代码示例"或"用表格对比这几个方案"。越清晰的指令，Claude 越能在搜索时精准过滤。
