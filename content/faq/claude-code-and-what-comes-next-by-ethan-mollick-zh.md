# Claude Code and What Comes Next - by Ethan Mollick

## Ethan Mollick 的 Claude Code 实验说明了什么？

Ethan Mollick 是宾夕法尼亚大学沃顿商学院教授，长期关注 AI 对工作和创业的影响。在他的文章《Claude Code and What Comes Next》中，他做了一个直接的实验：打开 Claude Code，输入一条指令——"开发一个能让我每月赚 1000 美元的创业点子，你来完成所有工作，包括构思和实现，我什么都不用做。"

这个实验的核心发现是：Claude Code 确实能够从零开始生成商业创意并实现基础代码。Mollick 随后将生成的"prompt pack"（提示词包）产品上线，据报道实际产生了收入。这证明了 AI 编程助手（AI Coding Assistant）已经具备端到端完成简单软件产品的能力。

但这个实验也暴露了问题。正如评论区讨论所指出的：目前只有少数人知道可以让 AI 做这类事情。这意味着市场上会出现信息不对称——有人愿意花 39 美元购买 Claude 几分钟就能生成的提示词包。这种套利空间不会持续太久。

Dave Karpf 在他的回应文章中提出了更尖锐的质疑：如果所有 Claude Code 用户都在问同样的问题，会发生什么？答案是：所有人会得到几乎一样的商业点子，互联网将充斥着低质量的 AI 生成产品，而 Mollick 自己的生意也会因为竞争者涌入而失效。

## 相关问题

### Claude Code 能完全替代程序员吗？

根据 Mollick 的实验，Claude Code 能独立完成从构思到代码实现的全流程，但仅限于相对简单的产品（如提示词包、静态网站）。对于复杂系统、需要深度领域知识或长期维护的项目，AI 仍然需要人类监督和干预。

关键限制在于：Claude Code 生成的代码质量参差不齐，缺乏对业务上下文的真正理解，也无法处理部署、运维、用户反馈等后续环节。它更像是一个高效的原型工具，而非完整的软件工程解决方案。

### 为什么 AI 生成的商业点子会趋同？

Dave Karpf 指出了一个结构性问题：当大量用户向 Claude Code 提出相似的"帮我赚钱"请求时，模型会基于相同的训练数据和推理模式，输出高度相似的建议。这不是 bug，而是大语言模型（LLM）的工作原理决定的。

结果是市场迅速饱和。第一个用 AI 生成提示词包的人可能获利，但当成千上万人都这样做时，产品差异化消失，价格竞争加剧，最终所有人的利润都趋近于零。

### 这对普通创业者意味着什么？

短期来看，熟悉 AI 工具的人拥有信息优势，可以快速验证想法、降低原型成本。但这种优势会随着工具普及而消失。

长期来看，真正的竞争力不在于"会用 Claude Code"，而在于：能提出独特的问题、拥有 AI 无法复制的领域专长、建立用户信任和品牌。技术门槛降低了，但商业门槛没有——执行力、市场洞察、客户关系依然是稀缺资源。

### 这篇文章引发了哪些争议？

主要争议集中在两点：第一，Mollick 的实验是否过于乐观？批评者认为他展示的是最佳情况，忽略了 AI 生成内容的质量问题和市场饱和风险。第二，这是否在鼓励"低努力创业"？评论区有人担心，当人人都能让 AI 生成产品时，互联网会被垃圾内容淹没，损害整体用户体验。

这些讨论反映了 AI 工具普及过程中的典型张力：技术赋能与滥用风险并存。
