# What's new in Claude 4.6

## Claude Opus 4.6 带来了哪些核心升级？

Claude Opus 4.6 是 Anthropic 在专家级推理能力上的重大突破。根据官方文档，这次更新主要集中在三个方向：更大的上下文窗口、更强的持续推理能力，以及面向复杂任务的新特性。

**上下文窗口大幅扩展**：Opus 4.6 默认支持 200K token 上下文窗口，同时提供 1M token（约 100 万词）的 beta 版本。这意味着你可以一次性输入整本技术手册或大型代码库，模型能完整理解并处理。最大输出 token 数也提升到了 128K，适合生成长文档、完整代码文件等场景。

**扩展思考（Extended Thinking）**：模型现在支持更长时间的"思考"过程，在回答复杂问题前进行更深入的推理。这对需要多步骤分析的任务特别有用，比如代码审查、方案设计、数据分析。

**智能体团队协作**：在 Claude Code 环境中，你可以组建多个智能体（Agent）协同完成任务。比如一个智能体负责代码生成，另一个负责测试，第三个负责文档——它们可以并行工作、相互配合。

**上下文压缩（Compaction）**：API 层面新增了压缩功能，Claude 能自动总结已处理的上下文，避免触及 token 上限。这让长时间运行的任务成为可能，模型不会因为对话过长而"失忆"。

据 Reddit 用户实测，在长流程任务中 Opus 4.6 的表现明显优于前代——约 90% 的复杂工作流场景中胜出。但对于简短提示，提升不明显。

### Claude Opus 4.6 和 Sonnet 4.6 有什么区别？

Opus 4.6 定位是旗舰级推理模型，专为复杂、长周期任务优化。Sonnet 4.6 则更注重速度和成本效益的平衡。根据 AWS 2026 年 2 月的公告，Sonnet 4.6 已在 Amazon Bedrock 上线，适合对延迟敏感、调用量大的生产场景。选型建议：需要深度分析、代码重构、长文档处理选 Opus；日常问答、快速响应选 Sonnet。

### 1M token 上下文窗口实际能装多少内容？

1M token 约等于 75-80 万英文单词，或 300-400 页技术文档。实际使用中，这意味着你可以把整个中型项目的代码库、完整的 API 文档集、或一本技术书籍完整输入。不过 1M 窗口目前处于 beta 阶段，生产环境建议先用 200K 验证稳定性。

### 智能体团队功能怎么用？

智能体团队（Agent Teams）目前在 Claude Code 中可用。你可以定义多个专注于不同任务的智能体，让它们协同处理复杂项目。典型场景包括：前端+后端+测试智能体并行开发、研究+写作+审校智能体协作生成报告。具体配置方式可参考 Anthropic 官方文档中的 Agent 部分。

### Opus 4.6 适合什么场景？

根据实测数据和官方定位，Opus 4.6 最适合这些场景：需要处理大量上下文的代码库分析、跨多个文件的重构任务、长篇技术文档生成、需要持续推理的研究任务。如果你的任务是简单问答或短提示，前代模型或 Sonnet 4.6 可能更经济。Opus 4.6 的优势在"大"和"复杂"——任务越长、越需要深度思考，收益越明显。
