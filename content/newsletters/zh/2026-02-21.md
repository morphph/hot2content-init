## 📝 技术实践

- 建 Agent 不想缓存的事？你在亏钱。Claude Code 团队的核心经验：提示缓存（Prompt Caching）几乎影响 Agent 系统的每个特性。缓存命中率直接决定成本和性能。[详情 →](https://x.com/trq212/status/2024638793719177291)

- 长时间运行的 Agent 总是跨上下文窗口翻车？Anthropic 工程团队从人类工程师的工作方式中找灵感，写了一篇关于如何设计更好的 Agent 运行框架（Harness）的实践指南。[详情 →](https://www.anthropic.com/engineering/effective-harnesses-for-long-running-agents)

- 跑分排行榜的差距可能是噪声。Anthropic 研究发现，基础设施配置差异就能让 Agent 编程基准测试（Coding Benchmark）波动好几个百分点——有时比排行榜上模型之间的差距还大。下次看跑分，记得打个折。[详情 →](https://www.anthropic.com/engineering/infrastructure-noise)

- Claude API 的 Bash Tool 深度讲解：持久化 Shell 会话，安装、编写、运行、调试在一个流程里完成，每条命令基于上一条的环境状态。[详情 →](https://x.com/adocomplete/status/2024738824564392333)

## 🎓 概念科普：提示缓存（Prompt Caching）

调用大模型 API 时，系统提示和工具定义这些每次都一样的内容，其实不需要每次重新处理。提示缓存就是把这些"不变的部分"存起来复用，下次调用直接跳过处理，既省钱又快。对于 Agent 系统尤其关键——一个 Agent 可能连续调用几十次 API，每次 90% 的输入都相同，不缓存等于白烧钱。
