**What is prompt caching and why does it matter for agents?**

When you send a prompt to an LLM, the model processes every token from scratch. Prompt caching lets the API remember the processed state of repeated prompt prefixesâ€”so if your system prompt stays the same across calls, you don't pay to process it again. For agents that make dozens or hundreds of API calls with similar context, this cuts latency and cost dramatically. The catch: you have to architect your prompts so the cacheable parts come first and stay stable.
