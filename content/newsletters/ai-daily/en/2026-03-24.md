## 🎓 MODEL LITERACY: Context Windows vs. Persistent Memory

Here's a concept that explains why half of today's trending repos exist: **context window limitations**.

Every LLM has a finite context window — the amount of text it can "see" at once. Even models with 200K token windows eventually hit the wall on long-running tasks. When an agent works for hours across many turns, earlier context gets lost. The agent forgets what it decided, repeats mistakes, or contradicts its own plan.

This is why projects like OpenViking (a context database), planning-with-files (persistent markdown plans), and RTK (token compression) all appeared on the same day. They're all attacking the same fundamental problem from different angles: memory compression, external storage, and structured planning documents that survive across context boundaries.

The short version: context windows are to agents what RAM is to computers. The ecosystem is inventing the equivalent of swap files, caches, and databases — all over again.
