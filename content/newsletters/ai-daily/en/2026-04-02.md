## 🎓 MODEL LITERACY: Source Maps

A **source map** (.map file) is a debugging tool that maps minified/compiled code back to original source files. They're essential for development but should never ship to production — they reveal your entire unobfuscated codebase. Bun (the JavaScript runtime Claude Code uses) generates source maps by default. The Claude Code leak happened because .map files weren't excluded from the npm package via .npmignore. Lesson: always audit what your build tools include in published packages.

## 🎯 PICK OF THE DAY: The KAIROS Architecture

The Claude Code leak's most fascinating revelation isn't the source code itself — it's KAIROS, an unreleased always-on autonomous agent mode referenced 150+ times in the codebase.

**How it works:** A tick loop injects periodic heartbeat messages. On each tick, the model decides: act or sleep. SleepTool lets the agent explicitly yield control, balancing API cost against cache expiration. A 15-second blocking budget auto-backgrounds slow commands. Append-only daily logs ensure full auditability.

**Why it matters:** KAIROS represents a paradigm shift from reactive (you prompt → it responds) to proactive (it monitors → decides → acts → notifies you). Three exclusive tools — push notifications, file delivery, and PR subscriptions — let it reach you even when the terminal is closed.

**autoDream** consolidates memory overnight using a forked sub-agent with read-only access, preventing corruption of the main context. The entire system is gated behind feature flags and not available publicly — but the code is complete and production-ready.

This is arguably the first fully-built "post-prompting" architecture from a major AI lab.
