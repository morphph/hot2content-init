## 🎓 MODEL LITERACY: Mixture of Experts (MoE)

You'll see "35B-A3B" in model names like Qwen3.5-35B-A3B. What does it mean?

**Mixture of Experts** is an architecture where a model has 35 billion total parameters but only activates 3 billion per inference. Think of it like a company with 35 specialists—for each task, only the 3 most relevant ones work on it. The router network decides which experts to activate.

**Why it matters:** You get the knowledge capacity of a 35B model with the inference speed and memory footprint closer to a 3B model. This is why Qwen 3.5 can run on consumer GPUs while competing with much larger dense models. The tradeoff: total model size on disk is still 35B, and training is more complex.

## 🎯 PICK OF THE DAY: OpenAI Container Pools

Agent developers have felt the pain: every tool call that needs code execution spins up a fresh container, adding seconds of latency. OpenAI's new container pooling in the Responses API changes this fundamentally.

Instead of cold-starting containers for every skill, shell command, or code interpreter call, the API now maintains a pool of warm containers that get reused across requests. The result is roughly 10x faster spin-up for agent workflows.

This is the kind of infrastructure improvement that compounds—faster containers mean agents can attempt more tool calls within the same latency budget, enabling more complex multi-step reasoning without user impatience. Combined with GPT-5.4 mini (which supports tool search and computer use), OpenAI is clearly optimizing the full stack for production agent deployment.

If you're building agents on the Responses API, this is a free performance upgrade with no code changes needed.
