## 🎓 MODEL LITERACY: Mixture of Experts (MoE)

**What it is:** A Mixture of Experts model contains many "expert" sub-networks but only activates a few for each input. Think of it like a hospital — you have specialists for cardiology, neurology, etc., but each patient only sees the relevant experts.

**Why it matters now:** Qwen3.5-35B-A3B is a hot example: "35B" total parameters, but "A3B" means only ~3B are active per token. You get near-35B quality with ~3B inference cost. This is why it's topping download charts — it runs fast on consumer GPUs while punching above its weight.

**The tradeoff:** MoE models use more memory (all experts must be loaded) but less compute per token. Great for inference speed, tricky for memory-constrained setups.

## 🎯 PICK OF THE DAY: Claude Code's /loop — Persistent Agents Are Here

Claude Code's `/loop` command is quietly one of the most significant agent features released this month. Instead of one-shot task execution, you can now tell Claude Code to run recurring tasks for up to 3 days.

The implications are big: continuous PR monitoring, daily Slack summaries, watching for build failures and auto-fixing them. This moves Claude Code from "tool you invoke" to "agent that works alongside you." Combined with the new Code Review feature (dispatching agent teams per PR), Anthropic is building toward always-on engineering agents that handle the tedious parts of software development while humans focus on architecture and design decisions.

If you use Claude Code, try: `/loop every 30 minutes check for new issues in my repo and triage them with labels`
