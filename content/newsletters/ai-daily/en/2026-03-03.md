## 🎓 MODEL LITERACY: Mixture of Experts (MoE)

This week's hottest models (Qwen 3.5, LFM2) all use **Mixture of Experts** architecture. Here's the idea: instead of activating all parameters for every input, the model has a "router" that selects only a subset of specialized "expert" sub-networks for each token. Qwen3.5-397B-A17B has 397 billion total parameters but only activates 17 billion per token — giving you near-400B quality at near-20B cost. This is why you'll see model names like "35B-A3B" (35B total, 3B active). MoE models are the reason open-source is rapidly closing the gap with closed models: you get frontier intelligence at inference costs small enough to run locally.

## 🎯 PICK OF THE DAY: Claude Code's Context Management Revolution

The biggest practical development this week isn't a new model — it's the tooling evolution around *how* we use AI for coding. Claude Code's upcoming /simplify and /batch skills, combined with the new auto-memory feature, represent a shift from "chat with AI about code" to "orchestrate AI agents that maintain quality at scale."

**/simplify** spawns parallel agents to analyze and improve code quality across your codebase, then creates a PR for review. **/batch** takes a migration task (like updating an API pattern across 50 files) and distributes it across dozens of isolated agent sessions, each with a fresh context window.

The auto-memory feature means Claude Code now builds a persistent understanding of your project across sessions — no more re-explaining your architecture every time you start a new chat.

Combined with Remote Control (manage sessions from your phone), Cowork scheduled tasks, and the Slack plugin, Claude Code is evolving from a coding assistant into a full development operations platform. The 4% of GitHub commits already authored by Claude Code is likely to accelerate significantly.
