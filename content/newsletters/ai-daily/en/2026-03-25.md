## 🎓 MODEL LITERACY: Supply Chain Attacks in the Age of AI Agents

A **supply chain attack** occurs when a malicious actor compromises a software dependency that gets pulled into your project. Today's LiteLLM incident made this viscerally real: a poisoned PyPI package (up for less than an hour) could exfiltrate SSH keys, AWS/GCP/Azure credentials, Kubernetes configs, API keys, crypto wallets, and more. Any project depending on LiteLLM — including DSPy and others — was vulnerable.

With AI agents now running `pip install` autonomously and managing complex dependency trees, the attack surface has exploded. As Karpathy noted, classical software engineering tells us dependencies are good ("we're building pyramids from bricks"), but this needs re-evaluation. Jim Fan pointed out that in the world of on-demand software, "every file that could go into context adds to the attack vector."

The package has been quarantined on PyPI. The takeaway: fewer dependencies, more vigilance, and serious investment in agent sandboxing.
[Read more →](https://x.com/karpathy/status/2036487306585268612)

## 🎯 PICK OF THE DAY: Claude Computer Use — The Desktop Is Now an API

Anthropic's release of full computer use in Claude Cowork and Dispatch marks a turning point in how we think about AI assistants. Boris Cherny, who created Claude Code, revealed that the same small Anthropic Labs team shipped MCP, Skills, Claude Desktop, and Claude Code — and those early computer use prototypes with Sonnet 3.6 felt "clunky and slow" but the vision was clear.

Now Claude can open apps, navigate browsers, fill spreadsheets — anything you'd do at your desk. Felix Rieseberg's Dispatch feature lets you kick off tasks and monitor them from your phone while you're away. Alex Albert put it simply: "The future where I never have to open up my laptop to get work done is becoming real very fast."

The safety model is noteworthy: Claude must request access to specific applications, and each action goes through permission gates. This is Anthropic's answer to the `--dangerously-skip-permissions` problem — which, coincidentally, also got replaced today with Claude Code's new auto mode.

We're moving from "AI writes code" to "AI uses your computer." The implications for workflows, accessibility, and automation are enormous.
