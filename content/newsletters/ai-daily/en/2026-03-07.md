## 🎓 MODEL LITERACY: What is Tool Search?

**Tool search** is a new capability in OpenAI's Responses API (and conceptually relevant across providers). Instead of loading all available tools into the context window upfront — which wastes tokens and hurts cache performance — tool search lets the model dynamically discover and load only the tools it needs at runtime. Think of it like lazy loading for AI tools: the model sees a lightweight index, picks what's relevant, and only then loads the full tool definitions. This is especially useful when you have dozens or hundreds of tools.

## 🎯 PICK OF THE DAY: The Infrastructure Noise Problem in AI Evals

Anthropic's new post on infrastructure noise deserves close attention. Their finding — that the *same model* can score several percentage points differently based on infrastructure configuration alone — has profound implications. When leaderboard gaps between top models are often 1-3%, this means infrastructure variance can be *larger than the actual performance difference*. Variables like network latency, sandbox configuration, timeout settings, and retry logic all compound. For teams building coding agents or relying on benchmarks for model selection, this paper is essential reading.

[Read more →](https://www.anthropic.com/engineering/infrastructure-noise)
