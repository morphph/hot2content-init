## 🎓 MODEL LITERACY: Infrastructure Noise in Benchmarks

When you see a leaderboard showing Model A beats Model B by 2 percentage points on a coding benchmark, you might assume that gap reflects genuine capability differences. Anthropic's new research challenges this assumption directly.

They found that simply changing infrastructure configuration—things like timeout settings, retry logic, container setup, and network conditions—can swing benchmark scores by several percentage points. That means the "gap" between top models on a leaderboard might be entirely explained by how the test was run, not how good the model actually is.

This is a form of measurement noise that the industry hasn't adequately accounted for. Just like you wouldn't trust a scale that gives different readings depending on which room you're in, benchmark results need to be interpreted with their infrastructure context. The takeaway: if the difference between two models is smaller than the infrastructure noise floor, you can't meaningfully rank them.

## 🎯 PICK OF THE DAY

**Claude Code Computer Use: The End of "Works on My Machine" for AI**

Today's Claude Code computer use launch isn't just another feature—it closes a fundamental loop in AI-assisted development. Until now, coding agents could write code but couldn't see what that code actually does when users interact with it. Claude could build your UI but had no way to click a button and verify the dropdown actually works.

Computer use in Claude Code changes this. From the CLI, Claude can now open your applications, navigate through UI flows, fill in forms, and test the software it just wrote. This means a single agent session can: write the code → build it → launch the app → click through the UI → identify visual bugs → fix them → verify the fix. No human in the loop for the mechanical testing parts.

The implications go beyond convenience. Felix Rieseberg (Anthropic) clarified that you control which apps Claude can access and whether it can look-only or also click and type. This granular permission model is critical—giving an AI full computer access is a security surface that needs careful boundaries.

Combined with today's auto mode launch for Enterprise (skip permission prompts safely) and the Codex-as-plugin arrival (use OpenAI's code review inside Claude Code), we're seeing the coding agent workspace converge into a single surface where multiple AI models collaborate, test their own output, and operate with increasing autonomy. The "vibe coding" era is evolving into "vibe testing."
