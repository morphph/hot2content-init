# GPT-5.4 Drops With Computer Use, Qwen 3.5 Floods HuggingFace, and Google Ships a CLI for Everything

OpenAI releases GPT-5.4 with built-in computer use, 1M token context, and a clever new tool search feature. Meanwhile, Qwen 3.5 dominates HuggingFace's trending page with models from 0.8B to 397B parameters. Google quietly ships an official CLI for Gmail, Drive, Calendar and more — complete with MCP server support.

## 🧠 MODEL

### GPT-5.4 Arrives With Computer Use and 1M Context
OpenAI shipped GPT-5.4 and GPT-5.4 Pro to the API today. The headline features: built-in computer use support (screenshot-based UI interaction), a 1M token context window with native compaction for long-running agents, and a new "tool search" feature that defers large tool surfaces until runtime — saving tokens and improving cache performance. GPT-5.4 Pro is available via the Responses API for problems that need more compute.

[Read more →](https://developers.openai.com/api/docs/changelog)

### Qwen 3.5: A Full Lineup From Tiny to Massive
Qwen's 3.5 series is dominating HuggingFace's trending page. The lineup spans 0.8B, 2B, 4B, 9B, 27B, 35B-A3B (MoE), 122B-A10B, and the flagship 397B-A17B — all multimodal (image-text-to-text). The 397B-A17B leads with 1.2K likes and 1.3M downloads. Unsloth has already shipped GGUF versions for local inference, and abliterated variants are popping up.

[Read more →](https://huggingface.co/Qwen/Qwen3.5-397B-A17B)

## 🎓 MODEL LITERACY: Tool Search

GPT-5.4 introduced **tool search** — a technique where instead of sending all available tools in every API call (which wastes tokens and hurts cache hit rates), the model defers discovering tools until it actually needs them at runtime. Think of it like lazy loading for function calling: the model only pulls in tool definitions when relevant to the current task. This reduces prompt bloat in agent systems with hundreds of tools, improves latency, and keeps your cache warm.
