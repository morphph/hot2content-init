## 📝 TECHNIQUE

**LiteLLM PyPI Supply Chain Attack: A Wake-Up Call**
Karpathy sounded the alarm on a poisoned LiteLLM package on PyPI that exfiltrated SSH keys, AWS/GCP/Azure credentials, Kubernetes configs, crypto wallets, and more. With 97M monthly downloads and transitive dependencies (including dspy), the blast radius was enormous. The poisoned version was up for only ~1 hour and was caught because the attacker's code crashed a machine by running out of RAM. Karpathy's takeaway: prefer "yoinking" functionality with LLMs over pulling in massive dependency trees. The package is now quarantined on PyPI.
[Read more →](https://x.com/karpathy/status/2036487306585268612)

**Jim Fan: "Claws Need Shells"**
Jim Fan expanded on the LiteLLM incident, noting that in the world of on-demand software, your entire filesystem is the attack vector. Every file that could go into context amplifies the risk. He predicts a blooming industry for "de-vibing" — audited Software 1.0 guardrails watching over agentic Software 3.0.
[Read more →](https://x.com/DrJimFan/status/2036494601750716711)

**Karpathy on LLM Memory Being Too Eager**
Karpathy noted a common issue with LLM personalization: a single question from months ago can keep surfacing as a "deep interest" in perpetuity. Models try too hard with memory, making it distracting rather than helpful.

## 🎓 MODEL LITERACY: Supply Chain Attacks in the Age of AI Agents

A **supply chain attack** happens when a malicious actor compromises a software package that other projects depend on. Instead of attacking you directly, they poison something you trust.

Think of it like food safety: you trust the restaurant, but if the ingredient supplier is compromised, every dish is affected.

In the LiteLLM case, a poisoned version of a popular Python package was uploaded to PyPI. Anyone who ran `pip install litellm` (or installed any package that depended on it) unknowingly executed code that stole their credentials.

This is especially dangerous in the AI era because:
- **Dependency trees are deep** — one package can pull in hundreds of others
- **AI coding agents install packages automatically** — less human review
- **Credentials are everywhere** — API keys, cloud tokens, SSH keys

Mitigations: pin dependency versions, use lockfiles, audit new installs, and consider "vendoring" critical functionality instead of importing it.
