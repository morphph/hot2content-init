## 🎓 MODEL LITERACY: Source Maps and Supply Chain Attacks

**What is a source map?** When developers compile or minify code (like TypeScript → JavaScript), a `.map` file is generated that maps the compiled code back to the original source. It's a debugging tool — never meant for production distribution. If you accidentally include it in your npm package (by forgetting to add it to `.npmignore`), anyone who installs your package can read your full original source code. That's exactly what happened to Claude Code this week.

**What is a supply chain attack?** Instead of attacking your code directly, attackers compromise a package you depend on. With axios getting 300M weekly downloads, poisoning it means potentially compromising millions of projects at once. This is why pinning dependency versions and auditing your lock files matters — and why coding agents that can scan for suspicious packages are becoming critical infrastructure.

## 🎯 PICK OF THE DAY: Computer Use in Claude Code

The arrival of computer use in Claude Code is a paradigm shift for agentic development. Previously, coding agents could write code but couldn't see what they built. Now Claude can:

- **Open applications** on your machine
- **Click through UIs** to test interfaces
- **Screenshot and verify** visual output
- **Iterate on design** based on what it actually sees

This closes the loop between "writing code" and "verifying it works" — the agent can now build a feature, launch the app, click around to test it, spot a visual bug, and fix it, all autonomously. The security model is thoughtful: you explicitly allow which apps Claude can access, and you choose whether auto-approval is enabled.

Combined with the new auto mode (which skips permission prompts for routine file operations), this moves Claude Code closer to a fully autonomous development environment. The question is no longer "can agents code?" but "how much of the development lifecycle can they handle end-to-end?"
