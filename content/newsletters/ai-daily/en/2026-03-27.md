## 🎓 MODEL LITERACY: What Are Agent "Skills" and Why Are They Everywhere?

If you've been watching GitHub trending this week, you've seen "skills" mentioned dozens of times. In the agent world, a **skill** is a reusable, self-contained capability that an AI agent can invoke — think of it like a plugin or function, but designed specifically for LLM-powered tools.

A skill might be "search the web," "create a PR," "analyze a CSV," or "write marketing copy." The key difference from traditional plugins: skills include natural language descriptions that help the agent understand *when* and *how* to use them, plus constraints on what they're allowed to do.

The explosion of skills marketplaces (5,400+ in one collection alone) mirrors what happened with npm packages, VS Code extensions, and smartphone apps — once you have a standard interface, an ecosystem emerges overnight. The open question is whether we'll also inherit the same problems: dependency hell, security risks, and a sea of low-quality entries drowning out the good ones.

## 🎯 PICK OF THE DAY: Anthropic's Eval Infrastructure Paper Changes How You Should Read Benchmarks

Anthropic's engineering team published ["Quantifying infrastructure noise in agentic coding evals"](https://www.anthropic.com/engineering/infrastructure-noise) — and it might be the most important thing you read this week if you make decisions based on model benchmarks.

The finding is straightforward but devastating: the way you configure your testing infrastructure (timeout settings, retry logic, hardware specs, network conditions) can swing benchmark scores by several percentage points. That's often larger than the gap between the top models on any leaderboard. In other words, a significant chunk of what we've been calling "model performance" might actually be "infrastructure performance."

This matters because companies, developers, and researchers are making real decisions — which model to use, which to fine-tune, which to invest in — based on benchmark margins that might be noise. If Anthropic is publishing this about their own evaluations, imagine how noisy third-party benchmarks are.
