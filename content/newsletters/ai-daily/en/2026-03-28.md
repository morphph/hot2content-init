## 🎓 MODEL LITERACY: Mixture of Experts (MoE)

You might have noticed models like "Qwen3.5-35B-A3B" — what does the "A3B" mean? It stands for **Active 3 Billion**: the model has 35 billion total parameters but only activates 3 billion on any given input.

This is the **Mixture of Experts (MoE)** architecture. Think of it like a company with 35 specialists, but only 3 are called into any meeting. Each input token gets routed to the most relevant "expert" sub-networks. The result: you get the knowledge of a much larger model at the speed and cost of a much smaller one.

Why it matters now: MoE is everywhere — from GPT-5.4 to Qwen 3.5 to NVIDIA's Nemotron Cascade. It's the key reason small-model performance keeps getting shockingly good. Bindu Reddy noted that small AI model usage is growing exponentially: "Haiku 4.6 is only 10% worse than Opus!"

## 🎯 PICK OF THE DAY: Claude Code Auto Mode

Anthropic's engineering blog on Claude Code auto mode is a must-read for anyone building or using AI coding agents.

**The problem:** Claude Code asks for permission before every file edit or command. Users approve 93% of prompts anyway, leading to approval fatigue. The alternative — `--dangerously-skip-permissions` — is exactly what it sounds like.

**The solution:** Auto mode uses model-based classifiers as a safety layer. Two layers of defense decide whether an action needs human approval. Their internal incident log reveals real-world horror stories: deleted remote git branches, leaked GitHub auth tokens, and accidental production database migrations — all from overeager AI, not malicious intent.

**Why it matters:** This is the frontier of AI safety in practice — not theoretical alignment debates, but real engineering decisions about when an AI coding agent should and shouldn't ask for permission. As agents get more autonomous, this approval-vs-autonomy tradeoff becomes the central design challenge.

[Read more →](https://www.anthropic.com/engineering/claude-code-auto-mode)
