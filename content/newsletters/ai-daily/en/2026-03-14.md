## 🎓 MODEL LITERACY: Mixture of Experts (MoE)

You'll see "120B-A12B" in model names like NVIDIA's Nemotron 3 Super. The "A12B" means only 12 billion parameters are **active** during any single inference, even though the model has 120 billion total.

This is **Mixture of Experts (MoE)** — the model contains multiple "expert" sub-networks, and a router decides which experts to activate for each token. You get the knowledge capacity of a huge model with the inference cost of a small one.

Qwen 3.5-35B-A3B uses the same trick: 35B total, only 3B active. That's why it runs on consumer GPUs while competing with much larger dense models.

**Bottom line**: When comparing models, check both the total parameter count *and* the active parameter count. A 120B MoE might be cheaper to run than a 30B dense model.

## 🎯 PICK OF THE DAY: Claude's 1M Context Goes Mainstream

The 1 million token context window going GA across Claude's model lineup is this week's most consequential release. Here's why it matters:

**For developers**: Claude Code now defaults to Opus 4.6 1M on paid plans. Combined with the new `/loop` command (persistent 3-day tasks) and Code Review (multi-agent PR analysis), this creates a genuinely new class of coding workflows. You can load entire codebases into context and have agents that understand your full project.

**For everyone else**: 1M tokens ≈ ~750K words ≈ roughly 10 full novels. You can now drop entire research papers, legal documents, or codebases into a single conversation. The practical limit shifts from "what fits in context" to "what's worth the cost."

**The competitive angle**: OpenAI's GPT-5.4 also ships with 1M context and compaction support. Google's Gemini has had long context for a while. We're entering an era where massive context is table stakes, not a differentiator — which means the real competition moves to *what models do with that context*.
