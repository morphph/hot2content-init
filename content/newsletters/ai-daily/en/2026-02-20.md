## ğŸ“ MODEL LITERACY: Prompt Caching

**What it is:** When you send a request to an LLM API, the system can cache the beginning of your request (the "prefix"). If your next request starts the same way, it reuses that cached computation instead of re-processing everything.

**Why it matters:** For agents like Claude Code that send dozens of API calls in one session, each call shares the same system prompt + tools + conversation history. Without caching, every call re-processes all of that from scratch. With caching, only the new part costs full price.

**The key insight:** It's a prefix match â€” everything must be identical from the start. Change one character in your system prompt? Cache miss. Reorder your tools? Cache miss. That's why Anthropic just launched automatic prompt caching â€” so you don't have to think about cache breakpoints anymore.
