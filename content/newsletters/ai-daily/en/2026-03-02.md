## 🎓 MODEL LITERACY: Mixture of Experts (MoE)

**Why is every new model suddenly "MoE"?**

A Mixture of Experts model is like having a team of specialists instead of one generalist. The model has many "expert" sub-networks, but only activates a few for each input token. A "router" decides which experts handle each token.

Why it matters now: Qwen3.5-35B-A3B has 35 billion total parameters but only uses 3 billion per inference. This means you get the knowledge capacity of a large model with the speed and memory of a small one. It's why these models can run on gaming PCs while matching frontier performance.

The trade-off? MoE models need more total memory to load (all experts sit in RAM), but they're dramatically faster and cheaper to run since only a fraction activates per token.

HuggingFace just published an updated deep-dive on MoE in Transformers this week.

## 🎯 PICK OF THE DAY: GGML Joins Hugging Face

The quiet acquisition that changes everything for local AI. When Georgi Gerganov created llama.cpp in 2023, it was a hobby project to run LLaMA on a MacBook. Three years later, it's the backbone of the entire local AI movement — every GGUF model, every local chatbot, every edge deployment traces back to this codebase.

The problem was sustainability. One person maintaining critical infrastructure used by millions is a familiar open-source story with a familiar ending. By joining Hugging Face, llama.cpp gets institutional resources while staying open-source and community-driven.

What this means practically: better integration with the HF ecosystem, faster support for new model architectures, and confidence that local AI inference won't depend on one developer's burnout threshold.
