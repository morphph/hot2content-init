## 🎓 概念科普：AI Agent 时代的供应链攻击

**供应链攻击**（Supply Chain Attack）是指攻击者通过污染你项目依赖的软件包来入侵你的系统。今天的 LiteLLM 事件让这个概念变得无比真实：一个被投毒的 PyPI 包（上线不到一小时）就能窃取 SSH 密钥、AWS/GCP/Azure 凭证、Kubernetes 配置、API Key、加密钱包等等。任何依赖 LiteLLM 的项目——包括 DSPy——都中招了。

AI Agent 现在会自主执行 `pip install` 并管理复杂的依赖树，攻击面已经爆炸式增长。Karpathy 说，传统软件工程告诉我们依赖是好的（「我们在用砖块建金字塔」），但这需要重新审视。Jim Fan 指出，在按需生成软件的时代，「每一个可能进入上下文的文件都是攻击向量」。

该包已在 PyPI 上被隔离。教训：减少依赖、提高警惕、认真投资 Agent 沙箱。
[查看详情 →](https://x.com/karpathy/status/2036487306585268612)

## 🎯 今日精选：Claude 电脑控制——桌面变成了 API

Anthropic 在 Claude Cowork 和 Dispatch 中上线完整电脑控制，标志着我们对 AI 助手认知的转折点。Claude Code 的创造者 Boris Cherny 透露，同一个小团队（Anthropic Labs）先后发布了 MCP、Skills、Claude Desktop 和 Claude Code——最早用 Sonnet 3.6 做的电脑控制原型「笨拙又慢」，但愿景从一开始就很清晰。

现在 Claude 可以打开应用、浏览网页、填写表格——你在电脑前能做的事它都能做。Felix Rieseberg 的 Dispatch 功能让你启动任务后可以离开电脑，在手机上远程监控。Alex Albert 一句话总结：「再也不用打开笔记本就能完成工作的未来，正在飞速变成现实。」

安全设计值得关注：Claude 必须请求特定应用的访问权限，每个操作都有权限门控。这是 Anthropic 对 `--dangerously-skip-permissions` 问题的回答——巧的是，今天 Claude Code 的 auto 模式也正式替代了它。

我们正在从「AI 写代码」走向「AI 用你的电脑」。对工作流、无障碍访问和自动化的影响是巨大的。
[查看详情 →](https://x.com/felixrieseberg/status/2036193240509235452)
