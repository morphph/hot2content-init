## 🎓 概念科普：Source Map（源码映射）

**Source Map**（.map 文件）是一种调试工具，能把压缩/编译后的代码映射回原始源文件。开发时很有用，但绝对不能带到生产环境——因为它会暴露你的全部未混淆源码。Claude Code 用的 Bun 运行时默认生成 source map，而团队忘了在 .npmignore 里排除 .map 文件，结果整个源码跟着 npm 包一起发出去了。教训：永远审计你的构建工具往发布包里塞了什么。

## 🎯 今日精选：KAIROS 架构深度解析

这次源码泄漏最值得关注的不是代码本身——而是 KAIROS，一个在源码中被引用 150+ 次的未发布自主 Agent 模式。

**工作原理：** 一个 tick loop 定期注入心跳消息。每次心跳，模型自己决定：干活还是睡觉。SleepTool 让 Agent 主动让出控制权，在 API 成本和缓存过期之间做权衡。15 秒的 blocking budget 自动把耗时命令丢到后台。Append-only 的日志系统确保完整审计能力。

**为什么重要：** KAIROS 代表了从被动（你提问 → 它回答）到主动（它监控 → 判断 → 行动 → 通知你）的范式转变。三个独占工具——推送通知、文件投递、PR 订阅——让它在终端关闭后也能主动联系你。

**autoDream** 利用一个只有只读权限的子 Agent 在夜间整合记忆，防止污染主上下文。整个系统被 feature flag 锁着，用户无法使用——但代码已经完整且可以投入生产。

这可能是主要 AI 实验室中第一个完整构建的"后提示"（post-prompting）架构。
