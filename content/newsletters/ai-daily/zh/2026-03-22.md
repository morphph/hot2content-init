## 🎓 概念科普：Mixture of Experts（MoE，混合专家）

你会在模型名里看到 "35B-A3B" 这样的标注，比如 Qwen3.5-35B-A3B。啥意思？

**MoE** 是一种模型架构：总共有 350 亿参数，但每次推理只激活 30 亿。打个比方，公司有 35 个专家，每个任务只派最相关的 3 个去干活，由一个"路由网络"（Router）决定派谁。

**为什么重要：** 你获得了 35B 模型的知识容量，但推理速度和显存占用接近 3B 模型。所以 Qwen 3.5 能在消费级 GPU 上跑，性能却能和大得多的密集模型掰手腕。代价是模型文件仍然是 35B 大小，训练也更复杂。

## 🎯 今日精选：OpenAI 容器池

做 Agent 开发的都体验过这个痛：每次工具调用需要跑代码时，都要冷启动一个新容器，白白浪费几秒钟。OpenAI 在 Responses API 里加的容器池彻底改变了这件事。

不再为每个技能调用、Shell 命令或代码解释器启动新容器，API 现在维护一个预热容器池供请求复用。结果是 Agent 工作流启动速度提升约 10 倍。

这种基础设施改进会产生复利效应——容器更快意味着 Agent 在同样的延迟预算内能尝试更多工具调用，支撑更复杂的多步推理而不让用户等得不耐烦。搭配支持工具搜索和 Computer Use 的 GPT-5.4 mini，OpenAI 显然在为生产级 Agent 部署做全栈优化。

如果你在 Responses API 上做 Agent，这是个不用改代码就能白嫖的性能升级。
