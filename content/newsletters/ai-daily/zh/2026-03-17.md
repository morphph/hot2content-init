## 🎓 概念科普：什么是子智能体（Subagents）？

子智能体是由"父 Agent"按需启动的专精 AI Agent，负责处理特定子任务。打个比方：你的主编码 Agent 遇到一个复杂项目，不用在一个上下文窗口里硬扛所有东西，它可以启动多个子 Agent——一个搞前端，一个搞后端，一个写测试。

核心优势：（1）**上下文隔离**——每个子 Agent 有独立干净的上下文窗口，避免上下文过长导致质量下降。（2）**并行处理**——多个子 Agent 同时干不同部分。（3）**可操控**——你可以单独调整某个子 Agent 的方向，不影响整体。

OpenAI 的 Codex Subagents 和 Claude Code 的多 Agent 能力都是这个模式的实现，正在成为复杂 Agent 编码任务的标准架构。

## 🎯 今日精选：阿里 SWE-CI 基准测试

本周最有分量的发现不是一个新模型，而是一种新的评估方式。阿里巴巴的 SWE-CI 基准测试不问 AI 能不能写代码，而是问它维护 233 天后代码还能不能跑。

结果很扎心：75% 的模型在维护过程中引入了回归 bug，技术债务像滚雪球一样累积直到代码库崩溃。只有 Claude Opus 4.5/4.6 的零回归率超过 50%。

这个基准之所以重要，是因为行业一直在为"快照式"基准优化——这道题现在能解吗？SWE-CI 问的是更难的问题：它能做一个靠谱的长期协作者吗？对大多数模型来说，答案是还不行。这个基准很可能会重塑我们评估和选择编码 Agent 的方式。
[查看详情 →](https://x.com/pvergadia/status/2033362617352556980)
