## 🎓 概念科普：混合专家模型（Mixture of Experts / MoE）

你可能注意到类似"Qwen3.5-35B-A3B"这样的命名——"A3B"是什么意思？它代表 **Active 3 Billion**（活跃 30 亿参数）：模型总共有 350 亿参数，但每次推理只激活 30 亿。

这就是 **混合专家模型（MoE）** 架构。可以理解为一家公司有 35 个专家，但每次开会只叫 3 个最相关的来。每个输入 token 会被路由到最合适的"专家"子网络。结果是：你用小模型的速度和成本，获得大模型的知识储备。

为什么现在很重要：MoE 已经无处不在——GPT-5.4、Qwen 3.5、NVIDIA Nemotron Cascade 都在用。这就是为什么小模型性能越来越好的关键原因。Bindu Reddy 说小模型使用量在指数增长："Haiku 4.6 只比 Opus 差 10%！"

## 🎯 今日精选：Claude Code Auto Mode

Anthropic 关于 Claude Code auto mode 的工程博客，做 AI 编码 Agent 的一定要看。

**问题**：Claude Code 每次编辑文件或执行命令都要申请权限。用户 93% 的时候都点同意，久了就变成"无脑审批"。另一个选项 `--dangerously-skip-permissions`——名字已经说明一切了。

**方案**：Auto mode 用模型分类器作为安全层，两层防御机制自动判断哪些操作需要人工审批。他们的内部事故日志记录了真实案例：误删远程 git 分支、泄露 GitHub auth token、对生产数据库执行迁移——全是 AI 过度积极导致的，不是恶意行为。

**为什么重要**：这是 AI 安全在实践中的最前沿——不是理论对齐辩论，而是真实的工程决策：AI 编码 Agent 什么时候该问、什么时候该自己干。随着 Agent 越来越自主，这个"审批 vs 自治"的权衡将成为核心设计挑战。

[查看详情 →](https://www.anthropic.com/engineering/claude-code-auto-mode)
