## 🔧 开发者工具

• **graphify：把任意代码库变成可查询的知识图谱** — 支持 Claude Code、Codex、OpenCode 等。指向一个代码/文档/论文文件夹，自动生成结构化知识图谱。我觉得这可能是 agent 工具链里最被低估的方向——agent 处理大代码库的瓶颈就是缺乏结构性理解，知识图谱恰好是对的抽象层。[查看详情 →](https://github.com/safishamsi/graphify)

• **caveman：用"原始人语法"砍掉 65% 的 token** — Claude Code skill，通过去掉冠词、填充词、冗余表达来压缩 agent 通信。"Why use many token when few token do trick。" 名字搞笑，省钱认真。如果你在大项目上经常撞 context 上限，配合 rtk 一起用效果更好。[查看详情 →](https://github.com/JuliusBrussee/caveman)

• **mempalace：号称历史最高分的 AI 记忆系统，免费** — 记忆是 agent 系统最难解决的问题之一——跨 session 忘记上下文、丢失决策记录、重复犯错。如果这个项目的基准测试结果靠谱，这对 agentic 工作流会是实质性推进。[查看详情 →](https://github.com/milla-jovovich/mempalace)

• **rtk：单 Rust 二进制文件，砍 60-90% 的 LLM token 消耗** — 上周就在 trending，继续火。零依赖，坐在你的开发命令和 AI agent 之间压缩输出。跑任何 agentic coding 工作流的，都应该试试。[查看详情 →](https://github.com/rtk-ai/rtk)

• **Vercel 发布 json-render：生成式 UI 框架** — 把 AI 生成的 JSON 渲染成 React 组件。如果你在做 chat 界面或 AI 驱动的 dashboard——模型决定 UI 结构，这是"模型输出 JSON"和"用户看到好看界面"之间缺失的那层。[查看详情 →](https://github.com/vercel-labs/json-render)

## 🎓 模型素养：什么是"知识图谱"，为什么 Agent 需要它？

当你让 AI agent 处理大型代码库时，它是一个文件一个文件读的——就像试图通过逐个读路牌来理解一座城市。知识图谱是一种关系结构化地图：这个函数调用了那个函数，这个模块依赖了那个模块，这个概念和那个概念相关。有了知识图谱，agent 不需要在原始文本中搜索，而是可以沿着关系推理——"如果我改了这个 API，下游哪些东西会挂？" 像 graphify 这样的新工具正在自动从代码和文档构建知识图谱，让 agent 拥有接近资深开发者头脑中那种结构性理解。
