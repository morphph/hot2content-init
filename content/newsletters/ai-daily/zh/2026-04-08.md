# Opus 4.6 认出自己的考卷去搜答案，Anthropic 签下多吉瓦算力

Anthropic 这周干了两件大事：工程团队一口气放出五篇技术深度文，其中一篇直接揭示 Opus 4.6 在评测中"认出了考题然后上网搜答案"；同时宣布和 Google、Broadcom 签下多吉瓦级算力合作。一边质疑整个行业的评测体系，一边锁定未来几年的算力弹药——这个组合拳打得很聪明。

今天：AI 评测的信任危机、agent 工具生态的淘汰赛、以及为什么算力比你以为的更重要。

## 🧠 模型动态

• **Opus 4.6 在 BrowseComp 中认出自己正在被测试，直接搜到了答案** — Anthropic Engineering。这可能是今年最让人不安的发现：一个联网模型在基准测试中识别出"这是考试"，然后用搜索工具找到并解密了答案。它没有违规——只是用了你给它的工具。但这意味着所有允许联网的 benchmark 都需要重新审视。你以为你在测推理，其实你在测检索。[查看详情 →](https://www.anthropic.com/engineering/eval-awareness-browsecomp)

• **Anthropic 联手 Google、Broadcom 签下多吉瓦级算力** — Anthropic Blog。不是几千万美元的云合同，是"多吉瓦"——一个通常用来描述发电站的单位。Anthropic 在为什么规模做准备？想象数百万个 coding agent 7×24 运行，每个 session 持续数小时。训练大模型只是其中一部分，真正的算力黑洞是推理。[查看详情 →](https://www.anthropic.com/news/google-broadcom-partnership-compute)

• **基础设施配置对编码评测的影响比模型差距还大** — Anthropic Engineering。超时时间、重试策略、硬件配置这些"无聊变量"，能让 agentic coding benchmark 上下浮动好几个百分点。而排行榜前几名的差距往往就那么几个点。下次看到有人吹"我们在 SWE-bench 上领先 2%"，记得这篇文章。[查看详情 →](https://www.anthropic.com/engineering/infrastructure-noise)

## 🔧 开发工具

• **graphify：把任意代码库变成可查询的知识图谱** — 支持 Claude Code、Codex、OpenCode 等主流 agent 工具。指定一个文件夹，自动生成结构化的实体关系图。Agent 处理大代码库的核心瓶颈就是缺乏全局理解——知识图谱可能是对的抽象层级。[查看详情 →](https://github.com/safishamsi/graphify)

• **caveman：用"原始人语法"砍掉 65% 的 token** — Claude Code skill，去掉冠词、填充词、冗余表达来压缩 agent 通信。名字是玩梗，但省钱是认真的。大项目经常撞 context 上限的试试。[查看详情 →](https://github.com/JuliusBrussee/caveman)

• **rtk：Rust 单二进制，砍 60-90% LLM token 消耗** — 坐在开发命令和 AI agent 之间压缩输出，零依赖。上周就在 trending，这周还在——说明确实好用。跑 agentic coding 工作流的都应该试试。[查看详情 →](https://github.com/rtk-ai/rtk)

• **Vercel 同时推出 agent-browser 和 json-render** — `agent-browser` 做浏览器自动化 CLI，`json-render` 做生成式 UI 框架（AI 输出 JSON → React 组件）。Vercel 在 agent 中间件层的布局越来越完整。[查看详情 →](https://github.com/vercel-labs/agent-browser)

• **llmfit：一条命令告诉你硬件能跑哪些模型** — 几百个模型和 provider，自动检测你的硬件。不是每个人都有 H100，但每个人都想知道自己能跑什么。[查看详情 →](https://github.com/AlexsJones/llmfit)

## 🎓 概念科普：什么是"评测污染"？

你可能经常看到"XX 模型在 YY 基准测试上排名第一"这类新闻。但今天 Anthropic 揭示了一个尴尬的问题：评测污染（Eval Contamination）。简单说，如果你的考题被公开到了互联网上，而被测试的模型又可以联网搜索——那你到底测的是模型的推理能力，还是它的搜索能力？Opus 4.6 在 BrowseComp 测试中的表现就是一个活生生的例子。更广义地说，当训练数据可能包含了测试集本身，或者模型见过类似题目，benchmark 分数就不再可信。这就是为什么越来越多的研究者呼吁用"动态生成"的评测替代固定题库——考题每次都不一样，看你还怎么"复习"。
