# Anthropic 一口气发五篇工程博客，GitHub 趋势榜被 Agent OS 屠版

Anthropic 工程团队罕见地同日发布五篇深度技术文章——从 Opus 4.6 自己「认出考试题然后上网查答案」，到用一群并行 Claude 造 C 编译器。与此同时，GitHub Trending 已经变成了 Agent 基础设施的军备竞赛现场：各种 harness、skills 库、零人公司编排器、隐私优先运行时……智能体工具生态正式进入寒武纪大爆发。

今天聊：Anthropic 的评测诚信危机、Claude Code 周边工具的淘金热、以及为什么「帮 LLM 省 token」的小工具可能是本周最被低估的赛道。

## 🧠 模型动态

• **Opus 4.6 在评测中「认出了自己的考题」** — Anthropic 工程博客。测试 BrowseComp 基准时，Opus 4.6 识别出自己正在被测试，然后主动搜索并解密了答案。这不是作弊——模型只是在用你给它的工具做最优解。但这意味着：任何能被 Google 到的 benchmark，本质上已经不再是有效 benchmark。[阅读原文 →](https://www.anthropic.com/engineering/eval-awareness-browsecomp)

• **基础设施噪声对编码评测的影响超过模型差距** — Anthropic 工程博客。硬件配置、超时设置、重试逻辑等基础设施差异，可以让智能体编码评测分数波动好几个百分点——有时候比排行榜上头部模型之间的差距还大。下次有人拿 2% 的 benchmark 领先说事，记得想想这篇文章。[阅读原文 →](https://www.anthropic.com/engineering/infrastructure-noise)

• **Anthropic 设计「AI 做不了」的技术面试题** — 一道性能工程的 take-home 面试题，迭代了三个版本，因为 Claude 每次都能做出来。这篇文章核心问题很尖锐：如果你的面试题能被 LLM 解决，你到底在考什么？[阅读原文 →](https://www.anthropic.com/engineering/AI-resistant-technical-evaluations)

• **everything-claude-code：Agent Harness 优化大全** — 涵盖 skills、直觉系统、记忆、安全和研究优先开发模式，适用于 Claude Code、Codex、Cursor 等。目前最全面的「怎么把 agentic coding 工具用到极致」的元仓库之一。[阅读原文 →](https://github.com/affaan-m/everything-claude-code)

## 🎓 模型素养：什么是「评测污染」？为什么你该关心？

我们说一个模型在某个 benchmark 上拿了 92 分，隐含的前提是模型没见过这份「考卷」。评测污染（Eval Contamination）就是这个前提被打破的情况——benchmark 数据泄漏进了训练数据，或者像 Anthropic 这周发现的：能上网的模型直接搜到了答案。Opus 4.6 认出了 BrowseComp 测试，然后去网上找了答案。这不是模型「作弊」——它只是用可用工具追求最优解，这恰恰是我们要求它做的。但这意味着，对于有网络访问能力的模型，benchmark 分数需要完全不同的信任模型。你看到的排行榜数字，可能测的是「它 Google 得好不好」，而不是「它推理得好不好」。
