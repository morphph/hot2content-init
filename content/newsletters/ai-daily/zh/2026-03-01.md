## 🎓 概念科普：Agent Harness（代理运行框架）

**是什么：** Agent Harness 是让 AI Agent 稳定长时间运行的"外壳"系统。它处理 Agent 的启动、监控、错误恢复、上下文管理、工具调用协调等基础设施问题——让 Agent 专注于任务本身。

**为什么今天值得关注：** 今天 GitHub Trending 上至少有 10 个 Agent Harness 相关项目（everything-claude-code、oh-my-opencode、nanobot、zeroclaw……）。这说明社区已经过了"演示能跑就行"的阶段，开始认真解决"生产环境怎么跑"的问题。Anthropic 今天那篇"Effective harnesses for long-running agents"也在讨论同样的主题——他们借鉴了人类工程师的工作模式来设计更有效的 Agent 运行框架。

## 🎯 今日精选：Anthropic 把 Agent 评估的坑都挖出来了

Anthropic 工程团队一周内连发五篇技术博客，系统性地拆解了 Agent 评估的方法论。其中最让我印象深刻的是这个发现：基础设施配置差异（超时设置、重试策略、沙箱环境）能让同一个模型在 benchmark 上波动好几个百分点——有时候比 leaderboard 上第一名和第二名的差距还大。这意味着很多"模型 A 比模型 B 强"的结论可能只是基础设施噪音。如果你在做 Agent 相关的产品或研究，这五篇文章是必读材料。
