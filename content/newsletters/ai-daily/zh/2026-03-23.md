• **Agent 评估没有银弹** — Anthropic Engineering。Agent 有用的那些能力——自主性、多步骤推理、工具使用——恰恰是让它们难以评估的原因。Anthropic 认为你需要组合多种评估技术，匹配被测系统的复杂度。实话说，大多数团队连单一指标都没做好，但方向是对的。 [查看详情 →](https://www.anthropic.com/engineering/demystifying-evals-for-ai-agents)

## 🎓 概念科普：什么是 Agent-Native 软件？

今天 HKUDS 发布的 CLI-Anything 主打一个概念："让所有软件变成 Agent-Native"。

**Agent-Native 是什么意思？** 传统软件是为人设计的——有按钮、菜单、表单。Agent-Native 的软件是为 AI Agent 设计的——它暴露结构化的接口（API、CLI、MCP 协议），让 Agent 能直接理解和操作，不需要"看屏幕"再点击。

**为什么你该关注：** 现在很多 Agent 工具在做的事情其实是"让 AI 模拟人类操作界面"——截屏、识别按钮、模拟点击。这很慢、很脆弱。真正的方向是软件本身提供 Agent 友好的接口。就像网站从"只给人看"到"提供 API 给程序调用"的转变一样，软件正在经历第二次类似的进化。
