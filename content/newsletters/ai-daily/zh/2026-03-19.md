## 🎓 概念科普：什么是子代理（Subagents）？

子代理是由"父代理"派生出来处理特定子任务的专门 AI 代理。可以理解为把工作委派给团队成员：

- **为什么重要**：单个代理的上下文窗口很快就会被填满。子代理让主代理保持专注，同时把调研、代码审查或测试委派到独立的上下文中。
- **怎么工作**：父代理决定何时启动子代理，给它一个具体任务，子代理独立工作——有时多个子代理并行运行。
- **实际例子**：在 Codex 里，你可以让一个子代理重构后端，另一个写测试，主代理负责协调整体计划。
- **关键限制**：子代理之间不能直接通信——它们向父代理汇报，由父代理统筹调度。

这个模式正在快速普及：Codex、Claude Code 和 LangGraph 现在都支持某种形式的多代理编排。

## 🎯 今日精选：Claude Cowork Dispatch

今天的亮点是 Dispatch——Anthropic 对"我希望 AI 在我离开电脑时继续干活"这个需求的回答。

**它是什么**：一个运行在桌面端的持久 Claude 对话，可以用手机随时发消息控制。布置一个复杂任务，出门，路上用手机查进度，回来看完成的工作。

**为什么重要**：这填补了同步 AI 聊天和异步 AI Agent 之间的空白。不需要服务器、云部署或命令行——只要 Claude Desktop 和你的手机。

**社区反应**：Ethan Mollick 说它覆盖了"我想用 OpenClaw 做的 90% 的事情"，而且安全风险更低。这个功能从研究预览到 100% Pro 用户推送只用了一天，说明 Anthropic 内部信心十足。

**更大的趋势**：方向很清晰——AI 助手正在从"坐在电脑前才能聊"进化为"永远在线的协作者"。Claude Dispatch、Codex 子代理、Claude Code 定时任务……Agent 时代比预想的来得更快。
