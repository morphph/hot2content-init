## 🎓 概念科普：混合专家模型（Mixture of Experts, MoE）

本周多个热门模型用了 **MoE** 架构，比如 NVIDIA 的 Nemotron-Cascade-2-30B-A3B 和各种 Qwen3.5 变体。

**什么是 MoE？** 传统模型每个 token 都要跑全部参数，MoE 则是把模型分成多个"专家"子网络，每个 token 只激活其中几个。一个门控机制（Gating）决定哪些专家来处理当前输入。

**为什么重要？** 比如一个 30B-A3B 模型，总参数 300 亿但每次只激活 30 亿——接近大模型质量，接近小模型速度和成本。这是行业把前沿模型做便宜的核心技术路线之一。

## 🎯 今日精选：Karpathy 的愿景——Agent 不只写代码，要能交付产品

Karpathy 最新的推文说了一个很简单但很深远的想法：未来的 Agent 不应该只是"帮你写代码"，而是要能处理从想法到上线部署的全流程——浏览各种服务、读文档、拿 API key、调试开发环境、部署到生产。

这个观点之所以引起共鸣，是因为每个 builder 都体会过：AI 写代码已经很强了，但真正卡脖子的是代码周围那一圈——支付接入、用户认证、数据库配置、域名解析、安全加固。Karpathy 认为整个 DevOps 生命周期必须代码化，并且要有 Agent 原生的操作接口。

我们正处在这件事"勉强能做到"的拐点。预计这会是 2026 年 coding agent 竞争的核心战场。
