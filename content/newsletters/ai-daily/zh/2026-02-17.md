## 🎓 概念科普：设计稿（Figma ↔ 代码）

**Figma + Claude Code 到底发生了什么？** 传统流程是：设计师在 Figma 里画，开发者在代码里重建——这中间有大量信息损失。

现在 Figma 让你把 Claude Code 生成的代码 UI 直接导入为**可编辑的设计稿**。这意味着：开发者向 Claude Code 描述一个 UI，得到可运行的 React 组件，然后推回 Figma 让设计师在视觉层面微调。**设计 ↔ 代码的鸿沟刚刚大幅缩小。而且是双向的。**

---

## 🎯 今日精选："AI Agent 读 Markdown"

目前最火的开发技巧看起来很简单：不要用自然语言描述你想要什么，而是给 AI 编码 Agent **结构化的输入**——ASCII 线框图、Markdown 规格书、甚至表格。

多个走红帖子展示了 Agent 解析结构化文本比解析散文要可靠得多。有开发者做了一个 ASCII 线框编辑器，30 秒画出布局，Agent 就能转换成像素级精确的代码。

教训是：**你的输入格式越好，输出质量就越高**。可以把它叫做编码 Agent 的 "Context Engineering"——而且它正在成为一项必备技能。
