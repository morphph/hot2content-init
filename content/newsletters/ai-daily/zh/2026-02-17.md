## 🎓 模型小课堂：设计帧（Figma ↔ Code）

**Figma + Claude Code 发生了什么？** 传统流程中，设计师在 Figma 创作，然后开发者在代码中重建一切——一个有损的交接。现在 Figma 支持将代码生成的 UI（来自 Claude Code）直接导入为可编辑设计帧。这意味着：开发者可以向 Claude Code 描述 UI，得到可运行的 React 组件，再推回 Figma 让设计师可视化调整。设计 ↔ 代码的鸿沟刚刚缩小了一大步。而且是双向的。

## 🎯 今日之选："AI Agent 读 Markdown"

当前最火的开发技巧出乎意料地简单：与其用自然语言描述需求，不如给 AI 编码 agent 结构化输入——ASCII 线框、markdown 规范，甚至表格。多个爆款帖表明，agent 解析结构化文本远比解析散文可靠。一个开发者做了个 ASCII 线框编辑器，30 秒产出布局，agent 再将其转化为像素级精准的代码。经验：你的*输入格式*越好，*输出质量*就越高。把它看作编码 agent 的"上下文工程"——正在成为必备技能。
