## 🎓 概念科普：Source Map 与供应链攻击

**Source Map 是什么？** 开发者把 TypeScript 编译成 JavaScript 时，会生成一个 `.map` 文件，用来将编译后的代码映射回原始源码。这是调试工具——绝不应该放进发布包里。如果你忘了在 `.npmignore` 里排除它，任何安装你包的人都能读到完整源码。这正是 Claude Code 这次的情况。

**供应链攻击是什么？** 攻击者不直接攻击你的代码，而是污染你依赖的软件包。axios 每周 3 亿次下载，一旦被投毒，意味着数百万项目可能受影响。这就是为什么锁定依赖版本、审计 lock 文件越来越重要——也是为什么能扫描可疑包的编程 Agent 正在成为关键基础设施。

## 🎯 今日精选：Claude Code 的 Computer Use

Claude Code 加入 Computer Use 是 Agent 开发范式的重大转变。此前编程 Agent 只能写代码但看不到效果，现在 Claude 可以：

- **打开应用** — 在你的机器上启动程序
- **点击 UI** — 像用户一样操作界面
- **截图验证** — 查看实际渲染结果
- **迭代设计** — 根据所见进行修改

这形成了从"写代码"到"验证效果"的完整闭环——Agent 可以自主完成：开发功能 → 启动应用 → 点击测试 → 发现 bug → 修复，全程无需人工介入。安全模型设计得很周到：你可以明确指定 Claude 能访问哪些应用，以及是否启用自动审批。

结合新的 Auto Mode（跳过常规文件操作的权限确认），Claude Code 正在向全自主开发环境演进。问题已经不再是"Agent 能不能写代码"，而是"开发生命周期中有多少环节能端到端交给 Agent"。
