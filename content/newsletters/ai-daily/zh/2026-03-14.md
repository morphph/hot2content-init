## 🎓 概念科普：混合专家模型（MoE）

你会在模型名字里看到"120B-A12B"这样的写法，比如英伟达的 Nemotron 3 Super。其中"A12B"表示每次推理时只有 120 亿个参数是**活跃**的，虽然模型总共有 1200 亿参数。

这就是**混合专家模型（Mixture of Experts, MoE）**——模型内部包含多个"专家"子网络，由一个路由器决定每个 token 激活哪些专家。你能获得大模型的知识容量，但只需要小模型的推理成本。

Qwen 3.5-35B-A3B 用的也是同样的技巧：总参数 350 亿，活跃参数仅 30 亿。所以它能在消费级显卡上跑，同时和大得多的密集模型竞争。

**划重点**：比较模型时，要同时看总参数量和活跃参数量。一个 120B 的 MoE 模型跑起来可能比 30B 的密集模型还便宜。

## 🎯 今日精选：Claude 百万上下文进入主流

Claude 百万 token 上下文窗口全面 GA，是本周最重要的发布。

**对开发者**：Claude Code 在付费套餐上默认使用 Opus 4.6 1M。结合新的 `/loop` 命令（最长 3 天持久任务）和 Code Review（多 Agent PR 审查），这创造了一个全新的编程工作流范式。你可以把整个代码库加载到上下文中，让 Agent 理解你的完整项目。

**对所有人**：100 万 token ≈ 约 75 万词 ≈ 大约 10 本完整的小说。你现在可以把完整的研究论文、法律文件或代码库放进一次对话中。实际限制从"上下文能装多少"变成了"值不值得花这个钱"。

**竞争格局**：OpenAI 的 GPT-5.4 同样支持 100 万上下文和压缩功能，Google Gemini 的长上下文也已经推出一段时间了。我们正在进入一个超长上下文成为标配而非差异化卖点的时代——真正的竞争将转向模型*如何利用*这些上下文。
