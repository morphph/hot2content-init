## 🎓 概念科普：混合专家模型（Mixture of Experts）

本周最热的模型（Qwen 3.5、LFM2）都用了 **MoE（混合专家）** 架构。核心思路：模型不是每次都激活全部参数，而是有一个"路由器"为每个 token 选择一小部分专家子网络来处理。比如 Qwen3.5-397B-A17B 总参数 3970 亿，但每个 token 只激活 170 亿——用接近 4000 亿的质量，花接近 200 亿的推理成本。这就是为什么模型名会写成"35B-A3B"（总量 35B，激活 3B）。MoE 是开源模型快速追赶闭源模型的核心原因：前沿智能，推理成本低到本地就能跑。

## 🎯 今日精选：Claude Code 的上下文管理革命

本周最有实战价值的进展不是新模型——而是 AI 编码工具链的进化。Claude Code 即将上线的 /simplify 和 /batch 技能，加上新的自动记忆功能，标志着从"跟 AI 聊代码"到"编排 AI Agent 军团大规模高质量执行"的转变。

**/simplify** 启动并行 Agent 分析整个代码库的质量，然后创建 PR 让你审核。**/batch** 把迁移任务（比如在 50 个文件中更新某个 API 模式）分发给几十个隔离 Agent 会话，每个都有全新上下文窗口。

自动记忆意味着 Claude Code 跨会话积累对你项目的理解——不用每次开新聊天都重新解释架构。

再加上 Remote Control（手机管理会话）、Cowork 定时任务和 Slack 插件，Claude Code 正从编码助手进化成完整的开发运维平台。GitHub 上已有 4% 的公开提交由 Claude Code 完成，这个数字大概率会加速增长。
