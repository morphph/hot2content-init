## 🎓 概念科普：什么是多 Agent 代码审查？

传统代码审查（Code Review）：一个人读你的 PR，（希望能）抓到 bug。多 Agent 代码审查：多个 AI Agent 并行工作，各自专注不同方面——一个查逻辑错误，一个看安全漏洞，一个评估性能影响——然后把发现汇总成一份统一的审查意见。

就像每个 PR 都有一个专家团队在审，而不是一个通才在看。Claude Code Review 就是这个思路，Anthropic 说它能抓到资深工程师都容易漏掉的 bug——在 AI 生成代码量爆发的当下，这一点尤其关键。

## 🎯 今日精选：Claude Code Review——AI 审查 AI 写的代码

元时刻到来了：AI 写了大部分代码，现在 AI 也来审查了。Anthropic 做 Code Review 的原因很直接：Claude Code 让他们工程师的代码产出增长了 200%，人工审查成了瓶颈。系统会给每个 PR 派出多个 Agent——不只是 lint 检查，而是深入的语义分析，找逻辑 bug、边界情况和架构问题。

为什么重要：当 AI 编码工具让代码产出翻了 10 倍，质量瓶颈就从"写"转移到了"审"。人可以用 AI 辅助写 10 倍的代码，但能审 10 倍的 PR 吗？Code Review 就是 Anthropic 的答案：让 AI 同时扩展审查能力。预计几个月内，主流 AI 编码工具都会跟进类似功能。
