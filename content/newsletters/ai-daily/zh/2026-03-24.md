## 📱 产品应用

• **Vercel Skills CLI：npx skills 一键装 Agent 技能** — Vercel 的第三款，开源 Agent 技能工具。三个项目加在一起，Vercel 的意图很明显：Agent 基础设施（浏览器）+ 展示层（Generative UI）+ 技能生态（Skills），一条龙全包。 [查看详情 →](https://github.com/vercel-labs/skills)

• **Google Workspace 终于有统一 CLI 了** — Drive、Gmail、Calendar、Sheets、Docs、Chat、Admin 全塞进一个命令行，从 Discovery Service 动态构建，还内置 AI Agent 技能。以前用一堆散装工具拼 Google Workspace 自动化的日子该结束了。 [查看详情 →](https://github.com/googleworkspace/cli)

## 📝 技术实践

• **长时间运行 Agent 的 Harness 设计** — Anthropic Engineering 观察了人类工程师如何在长任务中保持上下文连贯，然后把同样的策略搬到 Agent 上。核心发现：Agent 和人类在上下文碎片化时犯一样的错——忘记之前做了什么、重复已经走过的弯路。解法不是"加大上下文窗口"，而是系统化的上下文管理。 [查看详情 →](https://www.anthropic.com/engineering/effective-harnesses-for-long-running-agents)

• **Agent 评估没有银弹** — Anthropic 的另一篇工程博客。Agent 有用的那些能力（自主性、多步推理、工具调用）恰恰是让它们难以评估的原因。你需要组合多种评估策略，匹配被测系统的复杂度。大多数团队连一个 eval 指标都没做好，但方向是对的。 [查看详情 →](https://www.anthropic.com/engineering/demystifying-evals-for-ai-agents)

• **Manus 式持久化规划变成了 Claude Code 技能** — @OthmanAdi。把被 20 亿美元收购的 Manus 背后的工作流模式提取出来——Agent 的规划写在 Markdown 文件里，跨上下文窗口持久化。比起让 Agent 把计划存在脑子里然后忘掉，这个方法虽然简单但真的有效。 [查看详情 →](https://github.com/OthmanAdi/planning-with-files)

## 🎓 概念科普：什么是 Agent Harness？

Anthropic 今天的工程博客反复提到一个词：**Agent Harness**（Agent 运行框架）。

简单说，Harness 就是"Agent 的操作系统"——它不是 Agent 本身，而是 Agent 运行时的环境管理层。它负责：把长任务拆成可管理的片段、在上下文窗口之间传递关键信息、在 Agent 出错时提供恢复机制。

打个比方：Agent 是赛车手，Harness 是赛道+维修站+车队通讯。车手再厉害，没有好的后勤系统也跑不完耐力赛。Anthropic 的研究发现，Agent 失败的主要原因不是"模型不够聪明"，而是"运行框架没有帮它管好上下文"。这就是为什么今天 GitHub 上那么多 Agent 工具在做 Harness 层的创新。
