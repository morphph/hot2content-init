## 🎓 概念科普：Git Worktree（工作树）

**是什么？**
正常情况下一个 git 仓库只有一个工作目录。Worktree 可以让你同时在多个独立目录里 checkout 不同分支——共享同一个 `.git` 历史。

**为什么对 AI 编程重要？**
当你同时跑多个 AI agent 操作同一个 repo，它们会互相冲突（改同一个文件、抢 git 状态）。有了 worktree，每个 agent 有自己的目录和分支，各干各的，最后合并。

打个比方：以前是让所有 AI 程序员共享一个键盘，现在是给每人一张桌子。

**试试：** `git worktree add ../my-feature feature-branch`

## 🎯 今日精选：Claude Code 的并行革命

Claude Code 内置 worktree 支持不只是一个功能更新——它在改变编程范式。

**以前：** 开一个 Claude Code session，线性工作，等它做完。
**现在：** 同时开 5-15 个 session，每个有自己的 worktree，每个几分钟就 commit 一次。

Boris Cherny 每天都这么干。结果就是：你不再是「用 AI 写代码的程序员」，而是变成了**AI 程序员的经理**。你的工作变成了：
1. 把问题拆成可以并行的任务
2. 写清楚规格说明
3. Review 和 merge 结果

这和之前胡渊鸣说的「给 10 个 Claude Code 打工」一模一样——只不过现在有了官方支持的基础设施，不用再 hack 了。

核心启示：AI 辅助编程的技能天花板又提高了。瓶颈不再是「AI 能不能写好代码」——而是「你能不能分解问题、管理并行工作流」。
