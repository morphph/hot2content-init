# GPT-5.4 带着电脑操控能力来了，Qwen 3.5 刷屏 HuggingFace，Google 搞了个万能 CLI

OpenAI 发布 GPT-5.4，内置电脑操控、100万 token 上下文和全新的工具搜索功能。Qwen 3.5 系列从 0.8B 到 397B 全线铺开，直接霸占 HuggingFace 热榜。Google 悄悄上线了 Gmail、Drive、Calendar 的官方命令行工具，还自带 MCP 服务器。

## 🧠 模型动态

### GPT-5.4：内置电脑操控 + 百万上下文
OpenAI 今天发布了 GPT-5.4 和 GPT-5.4 Pro。三个重磅功能：内置 Computer Use（截屏式 UI 操控）、100万 token 上下文窗口（带原生压缩，适合长时间运行的 Agent）、以及全新的 Tool Search——不再把所有工具定义塞进每次请求，而是运行时按需加载，省 token 又提速。

[查看详情 →](https://developers.openai.com/api/docs/changelog)

Qwen 3.5 系列直接刷屏 HuggingFace。0.8B、2B、4B、9B、27B、35B-A3B（MoE）、122B-A10B，一直到旗舰 397B-A17B——全部支持多模态（图文理解）。397B 版本拿下 1.2K 赞和 130万次下载。Unsloth 已经出了 GGUF 量化版本，abliterated 去审查版也已经出现。

[查看详情 →](https://huggingface.co/Qwen/Qwen3.5-397B-A17B)

[查看详情 →](https://huggingface.co/zai-org/GLM-5)

### Gemini 3.1 Flash-Lite：追求速度的轻量版
Google DeepMind 发布 Gemini 3.1 Flash-Lite，定位高吞吐量场景下的快速推理。延续 Flash 系列"速度优先"的路线。

Aiden Bai 发布 React Grab——在页面上选中任意元素，然后告诉 Claude Code 或 Codex 要改什么。完全开源，`npx react-grab@latest` 即装即用。打通了可视化编辑和 AI 编程之间的鸿沟。

## 🎓 概念科普：Tool Search（工具搜索）

GPT-5.4 引入了 **Tool Search** 机制。传统做法是把所有可用工具的定义塞进每次 API 请求——工具多了就浪费 token，还影响缓存命中率。Tool Search 换了个思路：模型运行时按需搜索和加载工具定义，类似于"懒加载"。对于有几百个工具的 Agent 系统来说，这能显著减少 prompt 膨胀、降低延迟、提高缓存利用率。
