## 🎓 概念科普：Benchmark 里的基础设施噪声

你看到排行榜显示"模型 A 在编码 Benchmark 上比模型 B 高 2 个百分点"，直觉会觉得这代表 A 比 B 强。但 Anthropic 的新研究告诉你：先别急。

他们发现仅仅改变基础设施配置——超时设置、重试逻辑、容器环境、网络条件——就能让 Benchmark 分数波动好几个百分点。这意味着排行榜上模型之间的"差距"，可能完全是测试环境造成的，跟模型真实能力无关。

打个比方：你用一台秤称体重，换个房间秤的读数就不一样了，你还能信这个秤吗？Benchmark 也是同理。下次看 AI 排行榜，如果两个模型的差距小于基础设施噪声范围，那排名没有实际意义。
[查看详情 →](https://www.anthropic.com/engineering/infrastructure-noise)

## 🎯 今日精选

### Claude Code Computer Use：AI 开发进入「自我测试」时代

今天 Claude Code 的 Computer Use 功能不只是一个新 feature——它关闭了 AI 辅助开发中一个根本性的循环。以前编码 Agent 能写代码但看不到代码跑起来是什么样的。Claude 能帮你搭好 UI，但没办法点一下按钮验证下拉菜单是不是真的能用。

现在不一样了。从命令行开始，Claude 可以打开你的应用、在 UI 里导航、填表单、测试它刚写的软件。一个 Agent 会话就能完成：写代码 → 编译 → 启动应用 → 点击 UI → 发现视觉 bug → 修复 → 验证修复。机械性测试环节不再需要人类参与。

Felix Rieseberg（Anthropic）强调了安全设计：你可以控制 Claude 能访问哪些应用，以及它只能看还是也能点击和打字。这种细粒度权限控制很关键——给 AI 完全的电脑控制权是一个需要谨慎对待的安全面。

再结合今天企业版 Auto Mode 的上线（更安全地跳过权限提示）和 Codex 插件的到来（在 Claude Code 里用 OpenAI 的代码审查），我们看到编码 Agent 工作空间正在融合成一个界面——多个 AI 模型在同一个环境里协作、测试自己的输出、以越来越高的自主性运行。"Vibe coding" 时代正在进化为 "vibe testing" 时代。
[查看详情 →](https://x.com/claudeai/status/2038663014098899416)
